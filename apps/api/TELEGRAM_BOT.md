# Telegram bot — уведомления продавцам и админу

Каждый продавец может подключить бота к личному чату, группе или каналу и получать уведомления о новых заказах и смене статуса, а также управлять статусами из Telegram.

**Админ** может привязать один чат в настройках платформы — в него приходят **полные** уведомления по каждому заказу: новый заказ и любое изменение статуса (включая оплату) с полной информацией (хозир, сотувчи, манзил, тўлов, махсулотлар).

**Харидор** (фойдаланувчи Telegram орқали авторизация қилган, яъни `user.telegram_id` тўлдирилган): buyurtma расмийлаштирилганда «Buyurtmangiz qabul qilindi» хабар ва buyurtma ҳолати ўзгарганда «Buyurtmangiz yangilandi» хабар бот орқали юборилади. Тугма — «Mening buyurtmalarim» (сайтдаги buyurtmalar саҳифаси).

## Настройка

1. Создайте бота в Telegram через [@BotFather](https://t.me/BotFather): `/newbot`, укажите имя и username (например `MyShopUZSellerBot`).
2. Скопируйте выданный токен и **username бота** (без @). Добавьте в `.env`:
   ```
   TELEGRAM_BOT_TOKEN=123456789:ABCdefGHI...
   TELEGRAM_BOT_USERNAME=MyShopUZSellerBot
   ```
3. Примените миграции (если ещё не применены):
   ```bash
   npx prisma migrate deploy
   ```
4. Перезапустите API. Бот запустит long polling автоматически.

## Как пользоваться (продавец)

1. Зайти в **Sotuvchi panel → Sozlamalar** (Настройки продавца).
2. В блоке **«Telegram bildirishnomalar»** нажать «Ulash» и следовать инструкции:
   - Найти бота в Telegram (по username из BotFather) и отправить `/start` или `/link`.
   - Бот пришлёт одноразовый код (6 символов). Ввести этот код в форму на сайте и нажать «Ulash».
3. После привязки в выбранный чат будут приходить:
   - сообщения о новых заказах с кнопками «Tasdiqlash» / «Bekor qilish»;
   - сообщения об изменении статуса (в т.ч. после оплаты Click/Payme).
4. В сообщении о заказе можно менять статус кнопками: Tasdiqlash, Qayta ishlash, Yuborildi, Yetkazildi, Bekor qilish.
5. **Инлайн-кнопки в сообщении о заказе (продавец):** первый ряд — смена статуса; второй ряд — «📄 Batafsil» (подробности заказа в чат) и «📋 Buyurtmalar» (ссылка на панель).
6. **Инлайн-кнопки у админа:** в уведомлении о заказе — «📄 Batafsil» (полная сводка заказа в чат) и «📋 Buyurtmalar».
7. Команды бота:
   - `/start`, `/link` — код привязки + главное меню.
   - `/orders` — активные заказы (продавец) или последние заказы платформы (админ); под списком — кнопки «📄 {номер заказа» для перехода к деталям.
   - `/stats` — статистика: продавец — заказы, ожидающие, товары, доход; админ — платформа.
   - `/pending` — ожидающие заказы (продавец); у админа — счётчики модерации + кнопки-ссылки на «Tovarlar (moderatsiya)» и «Sharhlar» в веб-панели.
   - `/today` — за сегодня: продавец — свои заказы и сумма; админ — по всей платформе.
   - `/help` — справка и меню кнопок.

**Меню по ролям (отображается по-разному):**
- **Харидор (покупатель):** только «🛒 Do'kon (katalog, savatcha)», «📋 Mening buyurtmalarim», «❓ Yordam», «◀️ Asosiy menyu». В списке команд бота — только `/start`, `/shop`, `/orders`, `/help`.
- **Сотувчи (продавец):** кнопка Web App, затем Buyurtmalar, Statistika, Bugun, Kutilmoqda, Yordam, «🌐 Sotuvchi panel», «◀️ Asosiy menyu».
- **Админ:** то же + строка «📦 Moderatsiya (tovarlar)» и «💬 Sharhlar» (ссылки на модерацию), затем «🌐 Admin panel», «◀️ Asosiy menyu».

При нажатии любой кнопки **то же сообщение обновляется** (новое не создаётся). Кнопка «Asosiy menyu» возвращает к приветствию и меню. Длинные сообщения автоматически обрезаются до лимита Telegram (4096 символов). При открытии «Batafsil» по заказу показывается всплывающая подсказка «Yuklanmoqda…».

**Цветные кнопки** (Bot API 9.4): подтверждение и успех — зелёные (success), отмена — красные (danger), основные действия и ссылки — синие (primary).

Дополнительно: продавцу приходит уведомление о новом отзыве по товару; админу — о новом товаре и новом отзыве на модерации. В `.env` задайте `APP_URL` (например `https://myshop.uz`), чтобы в сообщениях были кнопки-ссылки на разделы сайта.

## Telegram Web App (Mini App)

В боте интегрирована **Web App** — мини-приложение, которое открывается внутри Telegram.

- **URL:** `{APP_URL}/telegram-app` (например `https://myshop.uz/telegram-app`). В API в `.env` должен быть указан `APP_URL` — адрес фронтенда (Next.js).
- **Как открыть:** в меню бота (кнопка «◀️ Asosiy menyu») отображается кнопка **«🛒 Do'kon (veb-ilova)»** — по нажатию открывается Web App. Если библиотека бота поддерживает `setChatMenuButton`, рядом с полем ввода в чате с ботом также появляется кнопка «Do'kon (veb)», открывающая то же приложение.
- **Содержимое:** страница приветствия, имя пользователя Telegram (если доступно), кнопка «To'liq saytni ochish» (открывает основной сайт в браузере) и ссылка на каталог.
- **Жесты:** вертикальные свайпы отключены (чтобы случайно не закрыть приложение); при выходе запрашивается подтверждение. Свайп от левого края экрана вправо выполняет переход «назад» (в приложении); кнопка «Назад» в шапке на главной странице — подтверждение и закрытие, на подстраницах — переход назад по истории.
- **Авторизация:** при открытии Web App из Telegram фронтенд автоматически отправляет `initData` на `POST /auth/telegram`. API проверяет подпись через `TELEGRAM_BOT_TOKEN`, находит или создаёт пользователя по `telegram_id` и возвращает JWT. Токен сохраняется в `localStorage`, пользователь считается авторизованным (каталог, корзина, заказы работают в контексте этого пользователя).
- **Production:** для работы Web App в Telegram нужен **HTTPS**. Домен приложения можно указать в [BotFather](https://t.me/BotFather) → ваш бот → Bot Settings → Menu Button (при необходимости).

## Подключение к группе или каналу

- **Группа:** добавьте бота в группу как участника, затем в настройках продавца отправьте боту в личку `/start` и привяжите аккаунт. После этого привяжите тот же бот к группе: в группе отправьте любое сообщение боту или добавьте его — для уведомлений в группу нужно будет один раз получить код, отправив боту в группе `/start`, и ввести этот код в настройках (текущая реализация привязывает один чат на магазин; для группы повторите шаг привязки, отправив `/link` в группе и введя новый код на сайте — последний привязанный чат и будет получать уведомления).
- **Канал:** добавьте бота в канал как администратора с правом публикации сообщений. Код привязки получите, переслав боту из канала сообщение или отправив в канале команду (если бот в канале поддерживает команды). Введите код в настройках продавца.

Тип чата (личный/группа/канал) сохраняется в поле `telegramType` (сейчас при привязке по коду сохраняется `PERSONAL`; при необходимости можно доработать определение типа по `chat_id` или через Bot API `getChat`).

## Kirish saytda Telegram orqali

На странице входа (Kirish) есть кнопка **«Telegram orqali kirish»**. Пользователь нажимает её → открывается ссылка на бота с параметром `start=login_<token>`. В Telegram пользователь нажимает **Start** (или отправляет команду автоматически) → бот привязывает его аккаунт к токену и создаёт/находит пользователя по `telegram_id`. Сайт опрашивает `GET /auth/telegram/verify?token=...` и при успехе выдаёт JWT и перенаправляет пользователя. Для работы ссылки в `.env` должен быть задан **TELEGRAM_BOT_USERNAME** (username бота без @).

## Telegramni mavjud hisobga ulash

Если пользователь уже зарегистрирован через **auth/register** (email + пароль), он может привязать Telegram к этому аккаунту:

1. Войти на сайт под своим аккаунтом.
2. В **Shaxsiy kabinet** (Profil) в блоке **«Telegram»** нажать **«Telegram ulash»**.
3. Откроется ссылка на бота с параметром `start=link_<token>`. В Telegram нажать **Start**.
4. Бот ответит, что аккаунт привязан. Страница кабинета опрашивает `GET /auth/telegram/verify?token=...` и при `status: "linked"` обновляет профиль — отображается «Ulangan».
5. После привязки пользователь получает уведомления о заказах в Telegram (новый заказ, смена статуса), как и при первом входе через Telegram.

Один Telegram-аккаунт можно привязать только к одному аккаунту сайта. Если этот Telegram уже привязан к другому пользователю, API вернёт ошибку.

## API (Web App — без JWT)

- `POST /auth/telegram` — вход/регистрация по данным Telegram Web App. Body: `{ "initData": "<query string из window.Telegram.WebApp.initData>" }`. Ответ: `{ accessToken, expiresAt, user }`; также выставляется cookie `refreshToken`. Требуется `TELEGRAM_BOT_TOKEN` в `.env`.
- `POST /auth/telegram/request-login` — запрос на вход через бота. Ответ: `{ token, loginUrl }` (например `https://t.me/BotUsername?start=login_xxx`). Требуется `TELEGRAM_BOT_USERNAME` в `.env`.
- `POST /auth/telegram/request-link` — **требуется JWT.** Запрос на привязку Telegram к текущему аккаунту. Ответ: `{ token, linkUrl }` (например `https://t.me/BotUsername?start=link_xxx`). Пользователь открывает ссылку в боте и нажимает Start; затем фронт опрашивает verify.
- `GET /auth/telegram/verify?token=xxx` — проверка: если пользователь нажал Start по ссылке **login_** — возвращает `{ accessToken, expiresAt, user }` и выставляет cookie; если по ссылке **link_** — возвращает `{ status: "linked" }` (JWT не выдаётся); иначе `{ status: "pending" }`.

## API (продавец, JWT)

- `POST /seller/telegram/link` — привязать по коду из бота. Body: `{ "code": "AB12CD" }`.
- `GET /seller/telegram` — статус привязки: `{ "connected": true, "telegramType": "PERSONAL" }`.
- `POST /seller/telegram/disconnect` — отвязать Telegram.

## API (админ, JWT)

- `POST /admin/telegram/link` — привязать админ-чат по коду. Body: `{ "code": "AB12CD" }`.
- `GET /admin/telegram` — статус: `{ "connected": true }`.
- `POST /admin/telegram/disconnect` — отвязать.

Админ привязывает тот же бот (тот же /start или /link), затем вводит код в **Admin panel → Platforma sozlamalari** (блок «Admin Telegram»). В этот чат уходят: полные уведомления по заказам; уведомление о новом товаре на модерации; уведомление о новом отзыве на модерации. При указании `APP_URL` в сообщениях отображаются кнопки-ссылки (Buyurtmalar, Moderatsiya, Sharhlar).

## База данных

- В таблице `users`: поле `telegram_id` (уникальный) — для автоматического входа через Web App; при первом входе создаётся пользователь с email `telegram_<id>@t.me`.
- В таблице `shops`: поля `telegram_chat_id`, `telegram_type`.
- В таблице `platform_settings`: поле `admin_telegram_chat_id`.
- Таблица `telegram_link_codes`: одноразовые коды для привязки (срок действия 15 минут).
- Таблица `telegram_login_tokens`: одноразовые токены для входа с сайта (поле `telegram_chat_id` заполняется ботом при `/start login_<token>`; срок действия 5 минут).

Если `TELEGRAM_BOT_TOKEN` не задан, бот не запускается, отправка уведомлений пропускается.
